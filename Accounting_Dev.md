# 記帳系統開發紀錄

## 專案資訊
- **專案名稱**：記帳資料解析與統計系統
- **技術棧**：Streamlit + SQLite + Python 3.11+
- **啟動指令**：`streamlit run app.py`

---

## 開發歷程

### Phase 1：初始建立
**建立基礎架構**

1. **requirements.txt** - 依賴套件
   - streamlit>=1.28.0
   - pandas>=2.0.0

2. **database.py** - 資料庫模組
   - 5 張資料表：persons, category_prices, expenses, pending_review, memos
   - CRUD 操作函式
   - 預設類別單價：早餐 $70, 中餐 $150, 晚餐 $150, 牛奶 $40, 點心/飲料/文具 $0

3. **parser.py** - 解析器模組
   - 行類型判斷（日期行、類別行、備忘）
   - 人名匹配（完全匹配 → 前綴匹配 → 模糊匹配）
   - 支援格式：`類別：人名`、`人名+金額`、`人名*次數`、`類別*次數：`

4. **app.py** - 主程式首頁
   - 系統狀態總覽
   - 類別單價設定
   - 最近記錄顯示

5. **pages/** - Streamlit 多頁面
   - `1_輸入.py` - 輸入頁面
   - `2_審核.py` - 審核頁面
   - `3_報表.py` - 報表頁面

### Phase 4：Bug 修復 - 牛奶金額計算錯誤
**問題：** 牛奶 1.5 份顯示 $90 而非 $60

**原因分析：**
- Parser 儲存：`amount = 40 × 1.5 = 60`（已乘過）
- Report 計算：`total = amount × quantity = 60 × 1.5 = 90`（又乘一次）

**修正：**
```python
# parser.py 修正前
amount = int(base_price * quantity)

# parser.py 修正後
amount = self.category_prices.get(category, 0)  # 儲存單價
```

**資料庫修正：**
```sql
UPDATE expenses SET amount = 40 WHERE category = '牛奶' AND amount = 60;
```

## 解析規則

### 支援格式
```
12/31（三）              ← 日期行（月曆模式下忽略）
早餐：小明、小華          ← 基本格式
點心：阿鵰點心+60         ← 帶金額
晚餐：傑森*2              ← 帶次數
牛奶*1.5：小明、小華      ← 類別帶預設次數
早餐：無                  ← 跳過
⚠️ 備忘內容              ← 備忘
```

### 人名匹配優先順序
1. 完全匹配詞庫
2. 別名完全匹配
3. 前綴匹配（「阿鵰點心」→「阿鵰」）
4. 模糊匹配（相似度 >= 0.6）
5. 無法匹配 → 送待審核
